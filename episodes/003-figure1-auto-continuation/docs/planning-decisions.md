# Episode 3 planning decisions

Episode 3 extends the Figure 1 reproduction from a Python-native continuation result into a backend comparison. The near-term backend is AUTO-07p; the longer-term comparison target is LOCA/Trilinos. The purpose is not to create a new model variant, but to run the same Bergner & Spichtinger equilibrium problem through independent continuation backends and compare normalized branch outputs.

## Backend-comparison goal

The comparison should answer whether independent continuation backends reproduce the same Figure 1 equilibrium branch family under the model semantics already used by Episode 2:

- Python-native continuation is the current reference implementation and source of branch outputs.
- AUTO should provide an independent continuation backend for the same residual equations and continuation coordinate.
- LOCA, when added later, should consume the same model semantics and branch-output schema rather than inventing a separate comparison format.

For Figure 1, all backends should target `p = 300 hPa`, `F = 1`, `T ∈ {190, 210, 230} K`, and `w ∈ [0.005, 2.0] m s^-1`.

## Terminology

**Backend-neutral branch schema**:
A tabular branch-output contract expressed in physical and comparison-oriented fields rather than solver-internal names. It should be usable for Python, AUTO, and later LOCA outputs.
_Avoid_: AUTO-only output format, fort.7 schema

**AUTO continuation branch**:
A Figure 1 equilibrium branch generated by AUTO-07p using the shared Fortran model core and normalized into the backend-neutral branch schema.
_Avoid_: independent AUTO sweep, unnormalized AUTO dump

**Backend comparison artifact**:
A curated output that compares branch states, residual norms, convergence metadata, and paper benchmarks across Python, AUTO, and later LOCA.
_Avoid_: screenshot comparison, backend-specific log only

## Continuation and state-coordinate decisions

- Continue in `log(w)` rather than raw `w` so the control coordinate matches Episode 2 and gives stable coverage over the logarithmic Figure 1 velocity axis.
- Report physical `w = exp(log_w)` in all normalized outputs.
- Use transformed state coordinates internally when needed for solver robustness, but normalize every branch point to physical `n`, `q`, and `s` for comparison.
- Keep `n > 0` and `q > 0`; invalid nonpositive cloud states are outside the unregularized residual domain.
- Treat `coefficients(env)` as independent of `w`; only the residual/control evaluation should replace `w` along a branch.

## Shared versus episode-local AUTO files

- Shared AUTO-07p model/backend assets that are reused across multiple episodes may live in the repository-level `auto/` directory.
- Episode-specific AUTO constants, run configurations, starts, diagnostics, generated files, and curated outputs for Figure 1 belong under `episodes/003-figure1-auto-continuation/auto/` or `outputs/`.
- Python scripts in this episode should orchestrate runs, normalize AUTO output, and write provenance rather than embedding reusable model formulas that belong in shared package/backend code.

## Provisional backend-neutral branch schema

Use CSV/Parquet-like tabular outputs with one row per branch point. The initial required fields are:

| Field | Type | Meaning |
| --- | --- | --- |
| `backend` | string | `python`, `auto`, or later `loca`. |
| `branch_id` | string | Stable branch name such as `figure1_T190K`. |
| `temperature_K` | float | Branch temperature in kelvin. |
| `pressure_Pa` | float | Environmental pressure. |
| `F` | float | Sedimentation scaling factor. |
| `N_a_m3` | float | Aerosol number concentration per cubic metre when fixed for the run. |
| `log_w` | float | Continuation control coordinate. |
| `w_m_s` | float | Physical vertical velocity, `exp(log_w)`. |
| `log_n` | float | Solver/comparison log ice-number coordinate when available. |
| `log_q` | float | Solver/comparison log ice-mass coordinate when available. |
| `n_kg_dry_air_inv` | float | Physical ice crystal number concentration per dry-air mass. |
| `q_kg_kg` | float | Physical ice crystal mass mixing ratio. |
| `s` | float | Saturation ratio. |
| `residual_norm` | float | Norm of the backend-normalized residual at the point. |
| `converged` | bool | Whether the backend reported convergence for the point. |
| `stability` | string/null | Optional backend stability label if available. |
| `backend_step_index` | integer | Original point order within the backend run. |
| `source_file` | string | Relative path to the raw or normalized source artifact. |

Recommended optional fields include backend-specific continuation labels (`auto_label`, `loca_step_id`), fold/bifurcation markers, solver iteration counts, row-scaling metadata, and machine/tool versions. Backend-specific fields should not be required by cross-backend comparison code.

## Expected comparison outputs

Initial Episode 3 outputs should be named and documented consistently enough to be regenerated:

- `outputs/figure1_auto_branches/branch_T190K.csv`, `branch_T210K.csv`, and `branch_T230K.csv` — normalized AUTO branches following the backend-neutral schema.
- `outputs/figure1_auto_branches/branches_all.csv` — combined normalized AUTO branch table.
- `outputs/figure1_auto_branches/run_metadata.json` — AUTO version, compiler/toolchain details, command provenance, model commit, tolerances, and source-file paths.
- `outputs/figure1_backend_comparison/backend_comparison_details.csv` — pointwise Python-vs-AUTO and benchmark comparisons after interpolating or matching by `log_w`.
- `outputs/figure1_backend_comparison/backend_comparison_summary.csv` and `.json` — per-variable and per-temperature error summaries.
- `outputs/figure1_backend_comparison/figure1_backend_comparison.png` — plot overlaying Python, AUTO, digitized paper curves, and Eq. 92--94 approximations where useful.

When LOCA is added, it should write the same branch-schema fields and extend the comparison tables with a `loca` backend rather than creating a separate schema.

## Testing and verification guidance

TASK-005 is complete and its guidance is available in `docs/testing.md`. Episode 3 implementation tasks should follow that document by layering tests:

1. fixed smoke tests for Figure 1 and Figure 4 environments;
2. Python-vs-Fortran coefficient, process-term, vector-field, and residual equivalence checks before branch-level comparison;
3. schema-invariant tests for normalized branch outputs;
4. deterministic curated comparison scripts for the scientific Figure 1 reproduction.

Optional external tools such as `gfortran` or AUTO should produce explicit pytest skips when unavailable; pure Python tests should remain runnable with `uv run pytest`.
